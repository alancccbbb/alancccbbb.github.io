🚀 Gridea + GitHub Pages 搭建博客完整教程

2025-04-22 Gridea ， GitHub Pages ， 网络实验

本教程将手把手教你如何使用 Gridea + GitHub Pages 搭建一个漂亮的静态博客，并配置 Gitalk 评论系统。适合新手快速上手。

✅ 一、准备工作

  1. 注册并安装 Gridea

    • 项目下载地址：https://github.com/getgridea/gridea
    • 支持 Windows / macOS / Linux

  2. 注册 GitHub 账号

    • 地址：https://github.com
    • 用于托管博客页面和评论内容

✅ 二、创建 GitHub 仓库

  1. 登录 GitHub，创建一个新的公开仓库：

    • 仓库名推荐为：yourusername.github.io（如你 GitHub 用户名是 alancccbbb，则仓库名应为 alancccbbb.github.io）
    • 不勾选 README 和其他初始化选项
    • 创建完成后记下仓库地址

  2. 启用 GitHub Pages：

    • 打开仓库 → Settings → Pages
    • Source 选择 master 分支 → / (root) → 保存
    • 生成的网址通常是：https://yourusername.github.io（如果自定义域名，可在Custom domain填入域名并在域名做好指向设置）

✅ 三、Gridea 博客设置

  1. 打开 Gridea，进入【远程】设置：
    • 远程类型：GitHub Pages
    • 域名：https://blog.lemom.cn
    • 仓库名称：alancccbbb.github.io
    • 分支名称：master
    • 用户名 & 邮箱：填写你的 GitHub 账号信息
    • Token：去 GitHub → Settings → Developer settings → Personal access tokens → 生成一个 Token，勾选 repo 权限
    • 填好保存，点击「初始化远程仓库」


  2. 设置博客信息：
    • 进入【设置】页面，填写博客名称、地址（如 https://blog.lemom.cn）、描述、作者等
    • 推荐配置 CNAME（见下文）

✅ 四、配置自定义域名（如 blog.lemom.cn）

  1. 登录你的域名服务商，添加以下 DNS 记录：

    • 类型：CNAME
    • 主机记录：blog
    • 记录值：yourusername.github.io（如 alancccbbb.github.io）

  2. 回到 GitHub 仓库 → Settings → Pages，添加自定义域名 blog.lemom.cn，并勾选「Enforce HTTPS」

  3. 在 Gridea 的远程-基础设置中CNAME 中填入：blog.lemom.cn

✅ 五、配置 Gitalk 评论系统

1. 🧰 准备工作

注册 GitHub 并登录

新建一个专用于存放评论的仓库，例如命名为：blog-comments

1. 注册 GitHub OAuth 应用

首先在Github的个人设置-Develop settings中注册一个OAuth Apps应用，在配置界面根据自己的博客填写相关信息。
或者直接前往 https://github.com/settings/applications/new：

  • Application Name：Gridea Blog
  • Homepage URL：https://blog.lemom.cn
  • Authorization Callback URL：https://blog.lemom.cn

完成后会生成：

  • Client ID
  • Client Secret

2. 开启 GitHub 仓库 Issues

进入博客仓库 → Settings → Features → 勾选 Issues

3. 配置 Gridea 评论系统（以 Gitalk 为例）

Gridea中的配置就比较简单了，只需要根据刚刚申请的OAuth Apps中所填的信息对应填写即可。

📌 注意事项

  • gridea-comments 仓库必须是公开的，否则 Gitalk 无法访问。

  • 仓库必须开启 Issues（进入仓库 → Settings → Features → 勾选 Issues）。

  • 博客每篇文章的标题将对应一个 GitHub Issue。

  • Gitalk 会自动按文章 path 或 title 创建 Issue 用于展示评论。

✅ 最后一步：部署 & 测试

写好文章后，点击「同步」部署博客

访问文章页 → 评论区 → GitHub 登录授权（首次使用需授权一次）

成功后可留言，评论将记录在 gridea-comments 仓库的 Issues 中
